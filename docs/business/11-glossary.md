# 11 Glossary

## Executive Summary

This glossary defines the core business and domain terms used across the business-analysis phase. The definitions are written for cross-functional readers including architects, analysts, product owners, and leadership stakeholders.

## Terms

| Term | Definition |
| --- | --- |
| AI Potential | The degree to which a use case can benefit from machine learning, optimization, computer vision, NLP, or related techniques |
| AIS | Automatic Identification System, a maritime tracking signal used to monitor vessel movement |
| Alert Latency | Time between data availability and generation of a decision-ready alert |
| Analyst Throughput | Volume of alerts, scenes, or cases an analyst can review in a defined time period |
| Capstone Scope | The bounded set of business problems chosen for the project’s initial implementation phases |
| Change Detection | Identification of meaningful differences between observations across time |
| Collision Avoidance | Operational activity to reduce risk from a possible conjunction between space objects |
| Conjunction | A close approach event between two orbiting objects that may require review or action |
| Data Freshness | How current data is when it becomes available for analysis or decision-making |
| Decision Support | Analytics or reporting that assists humans in making better decisions without removing human accountability |
| EO | Earth Observation, the use of satellites or related systems to observe the Earth’s surface or atmosphere |
| False Negative | A meaningful event that analytics fails to detect |
| False Positive | An alert or event flagged by analytics that is not actually meaningful |
| Ground Truth | Trusted reference information used to validate analytical outputs |
| Illegal Fishing Detection | Identification of suspicious fishing behavior in protected or regulated waters |
| KPI | Key Performance Indicator used to measure business, operational, or analytical success |
| Metadata Quality | Completeness, consistency, and usability of descriptive information about data assets |
| Mission Assurance | Confidence that a mission or operational objective will be completed safely and reliably |
| Mission Control | Operational function responsible for monitoring and directing mission activities |
| MVP | Minimum Viable Product, the smallest scope that can prove meaningful business value |
| Portfolio Value | The degree to which a use case strengthens the overall project as a platform demonstration |
| Predictive Maintenance | Anticipation of likely asset failure or degradation before breakdown occurs |
| RAG | Retrieval-Augmented Generation, a method that grounds generated responses in retrieved source material |
| Satellite Availability | Percentage of time a spacecraft or service remains operational |
| Satcom | Satellite communications services and the associated operational environment |
| Scope Drift | Expansion of project goals beyond the agreed boundaries of the current phase or release |
| Space Situational Awareness | Monitoring and understanding the orbital environment, including objects and potential hazards |
| Space Weather | Solar and geomagnetic activity that can affect spacecraft, communications, navigation, and power systems |
| Stakeholder | Any person or organization that uses, funds, regulates, or is affected by the platform |
| Triage Coverage | Share of important events reviewed within the required response window |
| Use Case | A specific business problem and decision workflow that the platform is intended to support |
